## 1. 项目目标与总体要求

本项目旨在完成一份高质量的**技术报告（10页）**及**PPT汇报**。我们要深入调研大模型安全领域的“提示词攻击”现象，剖析其底层原理，通过实战攻击案例验证漏洞，并编写代码实现基础的防御机制。

**核心原则：**
1.  **工作量导向：** 图表要多且精（Visio绘制），数据要详实，代码要跑通。
2.  **学术诚信：** 严禁直接使用大模型生成正文。所有观点、数据必须引用参考文献。
3.  **规范协作：** 所有产出物必须按照 Git 仓库结构提交。

## 2. 详细分工与任务包 (Work Breakdown)

请各位组员根据以下分工，认领任务文件夹，并将产出提交至仓库的 `task/` 目录中。

### Task A：文献综述
* **负责章节：** 1. 技术概述（800）2. 发展现状（1200）
* **具体任务：**
    1.  **概念界定：** 定义 Prompt Injection, Jailbreaking 等等（自选）。
    2.  **分类调研：** 梳理常见的攻击模式。
    3.  **案例分析：** 可以选取 1-2 个案例分析（反面案例）用来说明大模型提示词攻击的研究价值。
    4.  **现状梳理：** 调研 OpenAI, Anthropic 等厂商(自行挑选)的安全更新历程及相关学术论文。
* **交付物：**
    * `survey.md`: 包含文献综述的正文草稿（有图表用以突出大模型提示词攻击的重要性）和核心文献的 BibTeX 引用代码。
    * `xxxxx.pdf`: pdf版论文
* **其余要求：**
    * 尝试制作一个时间轴来表示发展过程.。
    * 可以使用图表进行说明。
    * 案例分析部分可以多写。

### Task B：底层原理与技术分析
* **工作区：** `task/02_principle/`
* **负责章节：** 3. 关键技术原理分析(1500)
* **具体任务：**
    1.  **机制分析：** 解释 Transformer 架构的 Token 预测机制，说明模型为何无法区分“指令”与“数据”。
    2.  **RLHF 局限性：** 分析为何经过 RLHF 对齐的模型仍然会被绕过。
    3.  **绘图：** 使用 Visio 绘制原理图（如：正常指令处理流程 vs 注入攻击处理流程）。
* **交付物：**
    * `principle.md`: 原理分析文字草稿和核心文献的 BibTeX 引用代码。
    * `xxxxx.pdf`: pdf版论文
* **其余要求：**
    * 关于原理图部分, 可以使用文字来说明/手绘/图片, Visio绘图会在后期统一格式后再制作. 
    * “定位并截取开源代码中（如 HuggingFace Transformers 库）关于 Attention Mask 或 Token Prediction 的核心函数片段，并对这几行代码做中文注释分析”。

### Task C：攻击实验
* **负责章节：** 4. 案例分析(1400)
* **具体任务：**

    3.  **记录数据：** 记录每个 Prompt 在不同模型上的响应情况（拒绝/成功/部分成功）。
* **交付物：**
    * `case_study.md`: 对于实验结果的少量分析和核心文献的 BibTeX 引用代码。
    * `experiment_data.xlsx`: 测试结果记录表。
    * `screenshots/`: 对话截图（作为报告中的核心案例证据）。
    * `xxxxx.pdf`: pdf版论文。
* **其余要求：**
    * 可以在 prompts 选取时参考论文.
